# 변경 가능성을 최소화하라

- 요약
  - 불변 클래스는 가변 클래스보다 설계하고 구현하고 사용하기 쉬우며, 오류가 생길 여지도 적고 훨씬 안전하다.
  - 불변 객체는 근본적으로 스레드 안전하여 따로 동기화할 필요 없다.
  - 클래스를 불변으로 만들기 위한 5가지 규칙
    - setter를 제공하지 않는다.
    - 클래스를 확장할 수 없도록 한다.(클래스 final 선언, )
    - 모든 필드 final 선언
    - 모든 필드 private 선언
    - 자신 외에는 내부의 가변 컴포넌트에 접근할 수 없도록 한다.(방어적 복사 수행)
